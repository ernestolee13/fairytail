# Third-party notices

Fairytail contains an adapted build-decision contract derived from Ponytail by
Dietrich Gebert. The adaptation is pinned to upstream commit
`16f29800fd2681bdf24f3eb4ccffe38be3baec6b` and is used under the MIT License.

Adapted portions:

- trace the real code flow and inspect callers before choosing a change;
- stop at the first working rung in the need, repository reuse, standard
  library, native platform, installed dependency, one-line, and minimum working
  diff ladder;
- preserve trust-boundary validation, data-loss prevention, security controls,
  accessibility basics, explicit requirements, and physical calibration;
- leave one smallest runnable check for non-trivial logic.

Fairytail-original portions include the canonical beginner fact layer,
profile-bounded analogy selection, English/Korean presentation, analogy
breakpoints, progressive disclosure, explanation packets, capability-aware
model routing, fallback behavior, and the three-arm measurement design.

No Ponytail visual asset is included, and this adaptation does not imply
endorsement by the Ponytail project or its author.
